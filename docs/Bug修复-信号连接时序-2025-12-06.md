## 🔍 根本原因分析

### 问题代码（修复前）
```python
class BranchConfigStep(QWidget):
    def _setup_ui(self):
        # 第87-88行：创建本地分支ComboBox并立即连接信号
        self.localBranchCombo = ComboBox(self)
        self.localBranchCombo.currentTextChanged.connect(self._validate_inputs)  # ❌ 过早连接
        layout.addWidget(self.localBranchCombo)
        
        # 第92-100行：加载分支列表并设置当前项
        branches = gitService.get_branches()
        if branches:
            branch_names = [b.name for b in branches if not b.is_remote]
            if branch_names:
                self.localBranchCombo.addItems(branch_names)
                current = gitService.get_current_branch()
                if current and current in branch_names:
                    self.localBranchCombo.setCurrentText(current)  # ⚠️ 触发信号
        
        # 第108-112行：创建远程分支LineEdit（在信号触发之后）
        self.remoteBranchEdit = LineEdit(self)  # ❌ 还没创建
        self.remoteBranchEdit.setPlaceholderText("...")
        self.remoteBranchEdit.setText("main")
        self.remoteBranchEdit.textChanged.connect(self._validate_inputs)
```

### 时序问题
1. **第88行**：`localBranchCombo`创建后立即连接`currentTextChanged`信号到`_validate_inputs`
2. **第100行**：调用`setCurrentText(current)`时触发`currentTextChanged`信号
3. **信号触发**：执行`_validate_inputs()`方法
4. **第124行**：尝试访问`self.remoteBranchEdit.text()`
5. **❌ 错误**：此时`remoteBranchEdit`还没创建（要到第108行才创建）

**核心问题**：信号连接时机过早，导致在控件未完全初始化时就触发了回调方法。

## ✅ 修复方案

### 修复代码（修复后）
```python
class BranchConfigStep(QWidget):
    def _setup_ui(self):
        # 第87-88行：创建本地分支ComboBox，暂不连接信号
        self.localBranchCombo = ComboBox(self)
        layout.addWidget(self.localBranchCombo)  # ✅ 不立即连接信号
        
        # 第91-99行：加载分支列表并设置当前项
        branches = gitService.get_branches()
        if branches:
            branch_names = [b.name for b in branches if not b.is_remote]
            if branch_names:
                self.localBranchCombo.addItems(branch_names)
                current = gitService.get_current_branch()
                if current and current in branch_names:
                    self.localBranchCombo.setCurrentText(current)  # ✅ 不会触发信号
        
        # 第108-111行：创建远程分支LineEdit
        self.remoteBranchEdit = LineEdit(self)
        self.remoteBranchEdit.setPlaceholderText("...")
        self.remoteBranchEdit.setText("main")
        layout.addWidget(self.remoteBranchEdit)
        
        layout.addStretch()
        
        # 第115-117行：所有控件创建完成后，再连接信号
        self.localBranchCombo.currentTextChanged.connect(self._validate_inputs)  # ✅ 延迟连接
        self.remoteBranchEdit.textChanged.connect(self._validate_inputs)
        
        # 第120行：初始验证
        self._validate_inputs()  # ✅ 此时所有控件都已创建
```

### 修复原则
**延迟信号连接**：所有UI控件创建完成后，再连接信号，确保回调方法执行时所有依赖的控件都已存在。

## 🎓 经验总结

### 最佳实践
1. **信号连接时机**：
   - ❌ 不要在控件创建后立即连接信号
   - ✅ 应在所有依赖控件创建完成后再连接信号
   
2. **UI初始化顺序**：
   ```python
   def _setup_ui(self):
       # 1. 创建所有控件
       self.widget1 = Widget1()
       self.widget2 = Widget2()
       
       # 2. 设置控件属性
       self.widget1.setText("...")
       self.widget2.setValue(...)
       
       # 3. 添加到布局
       layout.addWidget(self.widget1)
       layout.addWidget(self.widget2)
       
       # 4. 连接信号（最后）
       self.widget1.signal.connect(self.handler)
       self.widget2.signal.connect(self.handler)
   ```

3. **防御性编程**：
   - 在回调方法中检查控件是否存在
   ```python
   def _validate_inputs(self):
       if not hasattr(self, 'remoteBranchEdit'):
           return False
       # ...
   ```

### 类似问题排查
如果遇到`AttributeError: 'XXX' object has no attribute 'YYY'`：
1. 检查信号连接时机
2. 确认控件创建顺序
3. 验证回调方法的依赖关系
