# Gitora v1.3.0 发布说明

v1.3.0 引入可选的 AI 提交规划器。功能默认关闭，不内置模型，不改变 Gitora 原有的手动暂存、提交和推送流程。

## 新功能

- 连接用户已有的本地 Ollama 服务、远程 OpenAI 兼容 API（Chat Completions / Responses）或 Anthropic Messages API。
- 根据工作区改动生成可审阅的多组提交标题和可选正文。
- 按文件或 Git hunk 把改动规划为多个原子提交组。
- 在简化结果框中展示每组标题、正文和改动数量，确认后自动逐组提交并最终推送。
- 用户确认后按组创建多个 Commit，全部成功后一次推送。
- 读取最近提交标题，适配仓库的语言和 Conventional Commits 风格。

## 隐私与安全边界

- 远程 OpenAI 兼容 API、Anthropic Messages API 与非本机 Ollama 每次发送前显示分析范围、变更数量和预计字符数，并要求用户明确确认。
- Ollama 只调用用户填写的服务地址；仅本机回环地址直接发送，其他地址会按实际网络边界处理为远程发送。
- 图形界面输入的 API 密钥默认保存到当前用户的 Windows Credential Manager 或 macOS Keychain，不写入 AI 设置文件；环境变量仅作为自动化兼容回退，系统凭据优先。
- 模型没有 Git、Shell 或文件系统工具权限，只能返回受严格结构约束的建议。
- 所有计划在执行前检查结构、100% 覆盖和工作区指纹；代码块计划还会在隔离索引中试应用整份补丁。
- 不自动连续提交，不自动推送，不重写历史；所有提交仍由用户确认。

完整的数据发送范围和配置方式见 [AI 提交规划器使用指南](ai-commit-planner.md)。

## 评测状态

本版本包含 30 组真实 first-parent 历史回放样本，覆盖 60 个不重复提交及 Python、QML、测试、文档和混合改动，并为本地/远程提供方分别准备人工评价表。

回放框架的协议解析、覆盖、重复、补丁校验、耗时和失败类型可被记录；远程评测必须额外提供显式源码上传确认参数。由于发布准备环境未配置可用的本地或远程模型，本说明不提供也不推断模型接受率、标题质量或实际延迟结论。

## 已知限制

- 不随安装包分发 Ollama 或模型权重。
- 代码块拆分使用 Git hunk，不是语法树级拆分。
- 二进制、子模块、未解决冲突、符号链接和不完整差异不支持自动执行。
- 不自动运行仓库测试；模型生成质量不代表代码或测试正确。
- 高级上下文和网络限制暂未在图形界面开放。
- 原生凭据库仅支持 Windows 与 macOS；其他操作系统使用环境变量兼容路径。

## 升级说明

该功能不会迁移或改写现有仓库。首次使用时进入“设置 → AI 提交规划”，选择提供方、填写端点和模型名，通过连接检测后再启用。无需 AI 的用户可以保持默认关闭状态。
