## 🎯 问题描述

用户反馈：**远程SSH没有端口设置！**

对于使用非标准SSH端口（非22端口）的Git服务器，无法正确配置远程仓库URL。

---

## ✅ 解决方案

### 1. 添加SSH端口输入框

在 `RemoteInterface` 类中添加了SSH端口配置：

```python
# SSH端口配置（默认隐藏）
self.sshPortLabel = BodyLabel("SSH端口:", self)
self.sshPortEdit = LineEdit(self)
self.sshPortEdit.setPlaceholderText("默认: 22（仅SSH协议需要）")
self.sshPortEdit.setText("22")

# 初始隐藏，仅在检测到SSH URL时显示
self.sshPortLabel.hide()
self.sshPortEdit.hide()
```

### 2. 智能检测SSH格式

添加 `_on_url_changed()` 方法，自动检测URL格式：

```python
def _on_url_changed(self):
    """URL变化时检测是否为SSH格式"""
    url = self.urlEdit.text().strip()
    # 检测是否为SSH格式（git@开头或ssh://开头）
    is_ssh = url.startswith('git@') or url.startswith('ssh://')
    
    # 显示/隐藏SSH端口配置
    if is_ssh:
        self.sshPortLabel.show()
        self.sshPortEdit.show()
    else:
        self.sshPortLabel.hide()
        self.sshPortEdit.hide()
```

### 3. 自动转换URL格式

修改 `get_remote_info()` 方法，自动处理非标准端口：

```python
def get_remote_info(self) -> tuple[str, str]:
    """获取远程仓库信息（自动处理SSH端口）"""
    name = self.nameEdit.text().strip()
    url = self.urlEdit.text().strip()
    
    # 如果是SSH格式且端口不是22，需要修改URL格式
    if url.startswith('git@'):
        port = self.sshPortEdit.text().strip()
        if port and port != '22':
            # 将 git@host:path 转换为 ssh://git@host:port/path
            # 例如: git@github.com:user/repo.git 
            #    -> ssh://git@github.com:2222/user/repo.git
            if ':' in url:
                parts = url.split(':', 1)
                host_part = parts[0].replace('git@', '')
                path_part = parts[1]
                url = f"ssh://git@{host_part}:{port}/{path_part}"
    
    elif url.startswith('ssh://'):
        port = self.sshPortEdit.text().strip()
        if port and port != '22':
            # ssh://git@host/path -> ssh://git@host:port/path
            import re
            if not re.search(r':\d+/', url):
                url = re.sub(r'(ssh://[^/]+)(/)', rf'\1:{port}\2', url)
    
    return name, url
```

## 🎨 用户体验

### 智能显示/隐藏
- **HTTPS URL**: SSH端口配置自动隐藏
- **SSH URL** (`git@...` 或 `ssh://...`): SSH端口配置自动显示

### 默认值
- 默认端口：22
- 用户可以修改为任意端口（如：2222、8022等）

### 提示信息
```
🔑 支持HTTPS和SSH两种格式
💡 SSH端口仅在使用SSH协议时需要配置（如 git@...）
```

## 📝 使用示例

### 示例1：标准SSH端口（22）
```
输入URL: git@github.com:user/repo.git
SSH端口: 22（默认）
实际URL: git@github.com:user/repo.git（不变）
```

### 示例2：非标准SSH端口（2222）
```
输入URL: git@gitlab.company.com:user/repo.git
SSH端口: 2222
实际URL: ssh://git@gitlab.company.com:2222/user/repo.git
```

### 示例3：ssh://格式 + 非标准端口
```
输入URL: ssh://git@server.com/repo.git
SSH端口: 8022
实际URL: ssh://git@server.com:8022/repo.git
```

### 示例4：HTTPS（不受影响）
```
输入URL: https://github.com/user/repo.git
SSH端口: （自动隐藏，不显示）
实际URL: https://github.com/user/repo.git（不变）
```
