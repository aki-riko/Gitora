# SSH协议格式说明

**更新时间**: 2025-12-07 03:59  
**说明**: SSH和SSH://两种格式的区别

---

## 🔑 两种SSH格式

### 1. SSH (git@) - 简写格式 ⭐推荐
```
git@github.com:user/repo.git
```

**特点**:
- ✅ 最常用的SSH格式
- ✅ 简洁易读
- ⚠️ **只能使用标准端口22**
- ✅ GitHub、GitLab等主流平台都支持

**适用场景**:
- 使用标准SSH端口（22）
- GitHub、GitLab等公共平台
- 大多数Git服务器

---

### 2. SSH (ssh://) - 完整URL格式
```
ssh://git@github.com:22/user/repo.git
```

**特点**:
- ✅ 完整的URL格式
- ✅ **可以指定任意端口**（如2222、8022等）
- ✅ 更规范的写法
- ⚠️ 稍微冗长

**适用场景**:
- 使用非标准SSH端口
- 自建Git服务器
- 需要明确指定端口的场景

---

## 📝 使用建议

### 推荐选择

#### 标准端口（22）→ 选择"SSH (git@)"
```
输入:
  主机名: github.com
  路径: user/repo.git
  端口: 22

输出:
  git@github.com:user/repo.git  ← 简洁
```

#### 非标准端口（如2222）→ 自动使用"SSH (ssh://)"
```
输入:
  主机名: git.example.com
  路径: user/repo.git
  端口: 2222

输出:
  ssh://git@git.example.com:2222/user/repo.git  ← 完整格式
```

## 💡 自动处理逻辑

我们的代码会**智能处理**端口：

```python
if protocol == "git":  # SSH (git@)
    if port != "22":
        # 非标准端口，自动转换为ssh://格式
        url = f"ssh://git@{host}:{port}/{path}"
    else:
        # 标准端口，使用简写格式
        url = f"git@{host}:{path}"
```

**效果**:
- 选择"SSH (git@)" + 端口22 → `git@host:path`
- 选择"SSH (git@)" + 端口2222 → `ssh://git@host:2222/path`（自动转换）
- 选择"SSH (ssh://)" + 任意端口 → `ssh://git@host:port/path`

## 🎓 给新手的建议

### 不确定用哪个？
1. **GitHub/GitLab等公共平台** → 选择"SSH (git@)"
2. **自建Git服务器，非标准端口** → 选择"SSH (ssh://)"
3. **不想用SSH** → 选择"HTTPS"

### 最简单的选择
**99%的情况下，选择"SSH (git@)"就够了！**
