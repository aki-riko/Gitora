## 🔍 根本原因分析

### 问题定位

在 `BranchConfigStep` 的初始化代码中（第113行）：

```python
self.remoteBranchEdit.setText("main")  # ⚠️ 硬编码为 main
```

**问题**：
1. 配置向导**硬编码默认远程分支为 `main`**
2. 但是用户的远程仓库实际分支是 `master`
3. 代码检测到 `origin/main` 不存在，所以报错"没有任何分支"

### 为什么会这样？

近年来，GitHub 等平台将默认分支从 `master` 改为 `main`，但是：
- **老仓库**仍然使用 `master` 作为默认分支
- **新仓库**使用 `main` 作为默认分支
- **自建Git服务器**（如用户的 git.9li.life）可能使用 `master`

**硬编码 `main` 会导致老仓库配置失败！**

---

## 🔧 修复方案

### 智能检测默认分支

修改 `BranchConfigStep.__init__()` 方法，**智能检测**默认远程分支：

```python
# 智能检测远程分支：优先使用当前分支，其次是 master，最后是 main
default_remote_branch = "main"  # 默认值
current_branch = gitService.get_current_branch()
if current_branch:
    # 使用当前分支名作为默认远程分支
    default_remote_branch = current_branch
    logger.info(f"使用当前分支作为默认远程分支: {default_remote_branch}")
else:
    # 如果没有当前分支，检查是否有 master 分支
    branches = gitService.get_branches()
    local_branch_names = [b.name for b in branches if not b.is_remote]
    if 'master' in local_branch_names:
        default_remote_branch = "master"
        logger.info("检测到 master 分支，使用 master 作为默认远程分支")
    else:
        logger.info("使用 main 作为默认远程分支")

self.remoteBranchEdit.setText(default_remote_branch)
```

### 检测逻辑

**优先级**：
1. **当前分支** - 如果有当前分支，使用当前分支名（最合理）
2. **master 分支** - 如果本地有 master 分支，使用 master（兼容老仓库）
3. **main 分支** - 默认使用 main（兼容新仓库）
