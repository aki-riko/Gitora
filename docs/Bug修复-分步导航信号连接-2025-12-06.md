# Bug修复：分步导航自动关闭问题

**修复时间**: 2025-12-06 21:15  
**问题**: 添加SSH端口配置后，分步导航（GuideWindow）自动关闭  
**原因**: 信号连接时序问题  
**状态**: ✅ 已修复

---

## 🐛 问题描述

用户反馈：添加SSH端口配置功能后，初始化仓库的分步导航窗口会自动关闭。

---

## 🔍 问题分析

### 根本原因
在添加SSH端口配置时，将 `textChanged` 信号连接到 `_on_url_changed` 方法的时机不当：

**问题代码**：
```python
self.urlEdit = LineEdit(self)
self.urlEdit.setPlaceholderText("...")
self.urlEdit.setClearButtonEnabled(True)
self.urlEdit.textChanged.connect(self._on_url_changed)  # ❌ 过早连接
layout.addWidget(self.urlEdit)

# SSH端口配置（此时还未创建）
self.sshPortLabel = BodyLabel("SSH端口:", self)
self.sshPortEdit = LineEdit(self)
```

### 问题分析
1. `textChanged` 信号在 `urlEdit` 创建时就被连接
2. 但此时 `sshPortLabel` 和 `sshPortEdit` 还未创建
3. 当信号触发时，`_on_url_changed` 方法尝试访问尚不存在的控件
4. 导致异常或未定义行为，可能引起窗口关闭

---

## ✅ 解决方案

### 修改策略
**延迟信号连接** - 在所有UI元素创建完成后再连接信号

### 修复代码

#### 1. RemoteInterface（必填模式）
```python
# 创建所有UI元素
self.urlEdit = LineEdit(self)
self.urlEdit.setPlaceholderText("...")
layout.addWidget(self.urlEdit)

# SSH端口配置
self.sshPortLabel = BodyLabel("SSH端口:", self)
self.sshPortEdit = LineEdit(self)
# ... 创建完成

layout.addStretch()

# ✅ 所有UI元素创建完成后，再连接信号
self.urlEdit.textChanged.connect(self._validate_inputs)
self.urlEdit.textChanged.connect(self._on_url_changed)

# 初始验证
self._validate_inputs()
```

#### 2. OptionalRemoteInterface（可选模式）
```python
# 创建所有UI元素
self.urlEdit = LineEdit(self)
# ... 
self.sshPortLabel = BodyLabel("SSH端口:", self)
self.sshPortEdit = LineEdit(self)
# ... 创建完成

layout.addStretch()

# ✅ 所有UI元素创建完成后，再连接信号
self.urlEdit.textChanged.connect(self._on_url_changed)
```

## 🎓 经验教训

### 信号连接最佳实践
1. **延迟连接** - 在所有依赖的UI元素创建完成后再连接信号
2. **防御性编程** - 在信号处理方法中检查控件是否存在
3. **初始化顺序** - 创建 → 布局 → 连接信号 → 初始化数据

### 推荐模式
```python
def _setup_ui(self):
    # 1. 创建所有UI元素
    self.widget1 = Widget1()
    self.widget2 = Widget2()
    
    # 2. 添加到布局
    layout.addWidget(self.widget1)
    layout.addWidget(self.widget2)
    
    # 3. 连接信号（所有元素创建完成后）
    self.widget1.signal.connect(self._on_signal)
    
    # 4. 初始化数据
    self._init_data()
```
