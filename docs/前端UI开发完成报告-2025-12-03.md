## 功能1：Git冲突解决界面

### 新增文件
- `app/view/conflict_interface.py` (327行)
- `app/view/conflict_viewer_dialog.py` (68行)

### 核心功能
**ConflictInterface界面**:
- 自动检测合并状态（`gitService.is_merging()`）
- 显示冲突文件列表（`gitService.get_conflicts()`）
- 冲突文件卡片（ConflictFileCard）
  - 显示文件路径
  - 标记是否包含冲突标记（<<<<<<< >>>>>>>）
  - 提供三个操作按钮：
    - 查看冲突：打开ConflictViewerDialog
    - 使用我们的：调用`resolve_conflict_with_ours()`
    - 使用他们的：调用`resolve_conflict_with_theirs()`
- 顶部操作栏：
  - 刷新按钮
  - 中止合并按钮（调用`abort_merge()`）
- 状态卡片：
  - 无冲突：✅ 绿色图标
  - 有冲突：❌ 红色图标，显示冲突数量

**ConflictViewerDialog对话框**:
- 显示包含冲突标记的文件内容
- 语法高亮：
  - `<<<<<<<` 标记：蓝色
  - `=======` 分隔符：橙色
  - `>>>>>>>` 标记：绿色
- 只读显示，帮助用户理解冲突

### 集成方式
- 添加到主窗口导航栏（第4个导航项）
- 图标：FluentIcon.CANCEL
- 自动连接`gitService.statusChanged`信号刷新

## 功能2：Stash管理对话框

### 新增文件
- `app/view/stash_dialog.py` (304行)

### 核心功能
**StashDialog对话框**:
- Stash列表显示（StashItemCard）
  - Stash ID：`stash@{0}`
  - Stash消息：WIP on branch: message
  - 操作按钮：
    - 应用（Apply）：恢复但不删除
    - 恢复（Pop）：恢复并删除
    - 删除（Delete）：仅删除
- 顶部操作栏：
  - 消息输入框（可选）
  - 保存到Stash按钮：调用`stash_save(message)`
  - 刷新按钮
  - 清空所有按钮：调用`stash_clear()`
- 空状态提示：暂无Stash记录

**功能特点**:
- 支持带消息的Stash
- 删除操作带确认对话框
- 清空所有显示数量提示
- 操作完成自动刷新列表

### 集成方式
- 在repo_interface.py添加Stash按钮（顶部工具栏）
- 图标：FluentIcon.SAVE
- 点击打开StashDialog对话框

## 功能3：文件历史对话框

### 新增文件
- `app/view/file_history_dialog.py` (169行)

### 核心功能
**FileHistoryDialog对话框**:
- 左侧：提交历史时间线
  - 使用TimeLineWidget显示
  - 每个提交显示：消息、作者、Hash、日期
  - 支持选择提交（最多选2个）
- 右侧：文件内容显示
  - 查看单个版本：显示该版本文件内容
  - 对比两个版本：显示diff结果
- 操作按钮：
  - 查看此版本：调用`get_file_content_at_commit()`
  - 对比两个版本：调用`diff_file_between_commits()`

**功能特点**:
- 加载最近50次提交历史
- 使用`git log --follow`跟踪文件重命名
- 点击提交自动显示该版本内容
- 选中2个提交启用对比按钮
- 对比时自动排序（旧→新）

### 集成方式
- 在FileChangeCard添加右键菜单
- 菜单项："查看文件历史"
- 图标：FluentIcon.HISTORY
- 右键文件卡片打开FileHistoryDialog

## UI组件使用

### QFluentWidgets组件
- **Dialog**: 基础对话框（Stash、文件历史）
- **ScrollArea**: 滚动区域（冲突列表、Stash列表）
- **CardWidget**: 卡片容器（冲突文件卡、Stash卡）
- **TimeLineWidget**: 时间线（文件历史）⭐Pro组件
- **RoundMenu**: 圆角菜单（右键菜单）
- **LineEdit**: 输入框（Stash消息）
- **QTextEdit**: 文本显示（冲突内容、文件内容）
- **各种按钮**: PushButton、TransparentPushButton
- **各种标签**: StrongBodyLabel、BodyLabel、CaptionLabel

### 布局方式
- **QHBoxLayout**: 水平布局
- **QVBoxLayout**: 垂直布局
- **QSplitter**: 分割器（文件历史左右分割）
- **ScrollArea**: 滚动区域

## 交互流程

### 冲突解决流程
```
打开冲突界面
    ↓
自动检测合并状态（is_merging）
    ↓
显示冲突文件列表（get_conflicts）
    ↓
用户选择操作：
    - 查看冲突 → ConflictViewerDialog
    - 使用我们的 → resolve_conflict_with_ours
    - 使用他们的 → resolve_conflict_with_theirs
    - 中止合并 → abort_merge
    ↓
操作完成，自动刷新列表
```

### Stash操作流程
```
点击Stash按钮
    ↓
打开StashDialog对话框
    ↓
显示Stash列表（stash_list）
    ↓
用户操作：
    - 保存到Stash → stash_save(message)
    - 应用Stash → stash_apply(stash_id)
    - 恢复Stash → stash_pop(stash_id)
    - 删除Stash → stash_drop(stash_id)
    - 清空所有 → stash_clear()
    ↓
操作完成，自动刷新列表
```

### 文件历史查看流程
```
右键文件卡片 → 选择"查看文件历史"
    ↓
打开FileHistoryDialog对话框
    ↓
加载文件历史（get_file_history, 50条）
    ↓
显示时间线
    ↓
用户操作：
    - 点击1个提交 → 显示该版本内容
    - 选中2个提交 → 启用对比按钮
    - 点击对比 → 显示两个版本的diff
```
